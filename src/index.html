Introduction

Tortoise.js is a high-performance, object-oriented wrapper for the HTML5 Canvas API. It focuses on security, simplicity, and performance.

Why Tortoise?

Unlike raw Canvas context, Tortoise provides a controlled environment where elements are managed as objects, preventing common memory leaks, out-of-bounds rendering, and raw DOM manipulation conflicts.

Basic Implementation

try {
    const canvas = document.getElementById('game');
    tortoise.init(canvas);
    console.log("Engine started!");
} catch (err) {
    console.error("Initialization failed:", err);
}

API Reference - v1.0.0

Complete technical documentation for the Tortoise.js engine, based on the tortoise.js stable implementation.

1. Management Functions

Core functions to control the engine lifecycle and security layers.

tortoise.init(canvasElement)

Initializes the engine and applies the applySecurity layer. This prevents unauthorized use of raw 2D context methods like fillRect or drawImage outside the internal loop.

const canvas = document.getElementById('game');
tortoise.init(canvas);

tortoise.config(options)

Adjusts global rendering and performance settings:

  • fps: Target frames per second (Default: 60).
  • minFps: Triggers onError if performance drops below this value.
  • autoFrameClearing: Automatically clears the canvas every frame.
  • autoRemoveOutterElements: Removes elements that move outside the canvas boundaries.
  • forEachFrame: Callback executed every cycle before rendering.

State Control

tortoise.togglePause(): Toggles the pause state without clearing memory.

tortoise.kill(): Stops the loop, clears the canvas, and wipes all elements and classes.

tortoise.onError(callback): Sets a custom handler for security violations or performance drops.

2. Elements

Managing individual objects within the Canvas environment.

tortoise.elements.create(id)

Registers a new CanvasElement. IDs must be unique.

tortoise.elements.editId(oldId, newName)

Updates an element's ID globally and triggers the renamed event.

tortoise.elements.remove(id)

Permanently deletes the element and triggers the deleted event.

3. Styles & Properties

The editStyle(id, styles) method controls appearance, physics, and transformations.

Property Reference

Property Description Default
defaultMode Render mode: 'polygon', 'circle', 'text', or 'image'. 'polygon'
x, y Coordinates on the canvas. 0, 0
zIndex Stacking order (higher values render on top). 0
anchor Pivot point for rotation/positioning {x, y}. 0.5, 0.5
sides Number of sides for polygons (e.g., 4 for rectangles). 4
shadow Object: {color, blur, x, y}. blur: 0

4. Events

Tortoise.js uses an internal event propagation system.

tortoise.elements.onEvent(id, eventName, callback)

Supported Events: click, collision, deleted, styled, renamed.

Collision System (AABB)

Detects boundary overlaps and returns the hit direction.

tortoise.elements.onEvent('player', 'collision', (data) => {
    // data.collisionLocal returns 'top', 'bottom', 'left', or 'right'
    data.element1.preventWalkTrough(); // Resets position to prevent overlapping
});

5. Classes

Organize and manipulate groups of elements simultaneously.

tortoise.classes.create(className)

Initializes a new group array.

tortoise.classes.createElements(className, quantity)

Spawns multiple elements with unique IDs, automatically assigned to the specified class.

tortoise.classes.remove(className)

Deletes the entire class and removes all associated elements from the scene.

tortoise.classes.editName(oldName, newName)

Renames the class and updates the className property for all child elements.
